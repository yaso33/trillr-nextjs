أولاً: ميزات معالجة البيانات والعرض (Rendering)
Server Components (RSC): الميزة الافتراضية الآن؛ يتم تنفيذ المكونات على الخادم وإرسال HTML جاهز. هذا يقلل من حجم ملفات الـ JS التي يحملها المستخدم في "Trillr".

Client Components: تستخدمها فقط عندما تحتاج لتفاعل مباشر (مثل أزرار الإعجاب أو فتح الكاميرا لتصوير Reels) باستخدام 'use client'.

Streaming & Suspense: تسمح لك بإرسال أجزاء من الصفحة فور جاهزيتها. يمكن للمستخدم رؤية "الهيكل" (Skeleton) بينما يتم جلب البيانات الثقيلة في الخلفية.

Server Side Rendering (SSR): توليد الصفحة عند كل طلب، مفيد جداً لخلاصة المنشورات (Feed) لضمان الحداثة.

Static Site Generation (SSG): إنشاء الصفحات وقت بناء المشروع؛ تجعل الصفحات تفتح بسرعة البرق (مثل صفحة "من نحن").

Incremental Static Regeneration (ISR): تحديث الصفحات الساكنة بعد النشر دون الحاجة لإعادة بناء المشروع بالكامل. مثالي لصفحات الملفات الشخصية (Profiles).

ثانياً: ميزات تطوير المسارات والـ API
App Router: نظام توجيه يعتمد على المجلدات (Folders)، يدعم التصميم المتداخل (Nested Layouts) الذي يحافظ على حالة الشريط الجانبي (Sidebar) أثناء التنقل.

Server Actions: كتابة دوال تتعامل مع قاعدة البيانات (Supabase/Drizzle) مباشرة داخل المكونات دون الحاجة لـ Fetch API تقليدي.

Route Handlers: لإنشاء Webhooks أو API Endpoints مخصصة لاستخدامها من قبل تطبيقات أخرى.

Middleware: كود يعمل قبل وصول الطلب للصفحة؛ استخدمته أنت لتأمين المسارات وإضافة رؤوس الحماية (Security Headers).

ثالثاً: ميزات الأداء والتحسين التلقائي (Optimization)
Image Optimization: المكون <Image /> يقوم بتغيير حجم الصور وتحويلها لصيغة WebP وتأخير تحميلها (Lazy Loading) تلقائياً.

Font Optimization: تحميل الخطوط ذاتياً ودمجها في ملفات الـ CSS لمنع قفز النصوص (Layout Shift).

Script Optimization: التحكم في توقيت تحميل ملفات الـ JS الخارجية (مثل Google Analytics) لضمان عدم تأثر سرعة الموقع.

Metadata API: إدارة الـ SEO والعناوين والصور التعريفية (OG Images) لكل صفحة بشكل ديناميكي.

Fast Refresh: تجربة تطوير فورية؛ ترى التغييرات في الكود على المتصفح لحظياً دون فقدان حالة التطبيق.

رابعاً: ميزات الربط والبيئة البرمجية
Built-in CSS Support: دعم أصيل لـ CSS Modules و Tailwind CSS (الذي تستخدمه في Trillr) و Sass.

Typescript Support: دعم كامل وفحص دقيق للأنواع (Types) حتى في الروابط (Typed Routes).

Environment Variables: إدارة سهلة لملفات الـ .env للربط مع Supabase أو Render بأمان.

Automatic Code Splitting: تقسيم الكود تلقائياً؛ المتصفح يحمل فقط الـ JS المطلوب للصفحة الحالية، وليس المشروع بالكامل.

Instrumentation: تتبع أداء التطبيق واكتشاف الأخطاء في بيئة الإنتاج (Production) بشكل احترافي.
